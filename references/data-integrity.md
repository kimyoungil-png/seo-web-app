# Data Integrity Rules — 数値・事実の捏造禁止

## 大原則

> 入力データ、ユーザー提供情報、またはファクトチェック時に確認した信頼できる情報源に存在する事実だけを書く。

検索結果にない数値、機能、事例、評価、調査結果を、もっともらしさのために補わないでください。

## このアプリで利用できる根拠

1. Brave Search APIが返したWeb、Discussions、News、Videos、Suggestion
2. Web結果URLからPythonで抽出したタイトル、H2、H3
3. ユーザーが入力・編集したキーワード、Owned Site URL、CTA URL、構成案、独自性、OriginalityのAdditional Information、記事本文
4. Fact Check段階でBrave Search APIが取得し、選択中AIが比較評価した外部情報源

SERPのタイトルやスニペットは一次資料そのものではありません。記事本文で事実として断定する前に、必要に応じて一次資料で確認してください。

## 数値を扱う判定フロー

```text
数値を書きたい
  ↓
その数値は入力データまたは確認済み情報源に存在するか
  ├─ Yes → 単位・対象・時点を明記して書く
  └─ No  → 書かない。必要なら「(要出典)」と明記する
```

## 必須ルール

- 数値には単位、対象、時点を付ける。
- 「業界平均」「一般的には」など、根拠を示せない比較表現を使わない。
- Newsの公開時期や製品仕様など、変化しやすい情報は時点を明記する。
- Discussionsの投稿内容を一般的な事実として扱わない。ユーザーの困りごと・体験談として扱う。
- Videosのタイトル・スニペットだけで、動画内の結論や検証結果を断定しない。
- Suggestionは検索需要のヒントであり、質問への正解ではない。
- 取得できなかったカテゴリは「該当結果なし」「取得失敗」と明示し、推測で埋めない。
- 競合記事の文章をコピーしない。

## Owned Site URL / CTA URLの扱い

- URL文字列はリンク先を示す入力であり、そのサイトのサービス内容、価格、機能、実績、専門性、対応範囲を証明する根拠ではない。
- URLだけを見て、会社の強み、受賞歴、導入実績、料金、無料提供、成果保証を作らない。
- Owned Site URLは、読者が追加情報を確認するための案内先として自然に利用する。
- CTA URLが空欄の場合、仮URL、リンク、CTAプレースホルダーを生成しない。
- CTA URLがある場合も、記事文脈に合う1か所だけで使用し、根拠のない効果や保証を付けない。
- リンク先の内容を確認していない場合は、アンカーテキストも断定的にしすぎない。

## OriginalityのAdditional Informationの扱い

- Additional Informationは、参考URL、意見、現場知見、補足条件などのユーザー提供コンテキストである。
- ユーザーの意見や仮説は、客観的な事実、一般的な評価、第三者の合意として断定しない。
- URL文字列だけでは、リンク先の本文、サービス内容、価格、機能、実績、調査結果を確認したことにならない。
- URLだけが入力されている場合、リンク先の内容を推測せず、必要ならリンク候補としてのみ扱う。
- 数値、実績、引用、固有の事例を記事へ入れる場合は、入力内に根拠があるか、別途確認済みである必要がある。
- 追加情報がSERP根拠や確認済み事実と矛盾する場合は、断定を避け、追加確認が必要であることを示す。
- 個人情報、秘密情報、APIキーなどを記事本文へ転記しない。

## H2/H3抽出データの扱い

- `fetch_error`のあるページは頻出見出し集計に含めない。
- インジェクション疑いが検出されたページはH2/H3を空にして分析対象から除外する。
- 見出しが取得できない場合でも、Braveが返したタイトル・スニペットはWeb結果として表示できる。
- 頻度は取得に成功したページ群だけを母数として解釈する。

## Fact Check

- 選択中AIは記事から検証可能な主張を抽出するが、この段階では真偽を決めない。
- 外部情報源の取得にはBrave Web Searchを使い、AI固有のWeb検索機能は使わない。
- 同一ドメインの重複を避け、各主張につき可能な限り3つ以上の独立した高品質な情報源を確認する。
- AIが出力するURLは、Brave Searchが実際に返したURLに限定する。
- True / Minor Errors / Needs Double-Checking / Falseの判定根拠を示す。
- 独立情報源が3件未満の場合、`True`をそのまま採用せず`Needs Double-Checking`として扱う。
- 情報源が不足する場合は「不明」「追加確認が必要」と明記する。
- 古い情報は、現在の有効性にどう影響するか説明する。
- 途中結果を保存して再開しても、別記事・別モデル・別検索地域の証拠を混在させない。

## 完了前チェック

- [ ] 入力にない数値や事例を追加していないか
- [ ] 変化しやすい情報に時点を付けたか
- [ ] 取得失敗や欠損を隠していないか
- [ ] Discussions、Videos、Suggestionを事実そのものとして扱っていないか
- [ ] 競合文をコピーしていないか
- [ ] URLだけを根拠にOwned Siteの機能・実績・価格を作っていないか
- [ ] CTA URLが空欄なのにリンクや仮URLを生成していないか
- [ ] Additional Information内の意見やURLを、未検証の客観的事実として扱っていないか
